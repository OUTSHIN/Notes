# UNIX
> 假定安装了编译器cc，file1.c和file2.c是含源代码的文件

- 编译两个文件并生成名为a.out的可执行文件
```
cc file1.c file2.c
```
另外，还生成了file1.o和file2.o的目标文件。如果改动了file.c ，而file2.c不变，可以使用以下命令编译第一个文件，并与第二个文件的目标代码合并：
```
cc file1.c file2.o
```

# Linux
> 假定安装了GNU的编译器gcc ...

- 编译生成a.out可执行文件
```
gcc file1.c file2.c
```
同UNIX ....

# DOS命令行编译器
- 与UNIX类似，只是名称不同。
- **区别**：对象文件的扩展名是.obj，而不是.o ，一些编译器生成的不是目标代码文件，而是汇编语言或者其他特殊代码的中间文件

# 使用头文件
> 把函数原型放在头文件中，就不用在每次使用函数文件时都写出函数的原型只需要在源文件使用#include指令包含头文件即可
